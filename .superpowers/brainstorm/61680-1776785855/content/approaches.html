Tech stack — 3 approaches

Pick the one that fits best. I've marked my recommendation.

A

⭐ Next.js (React) — Recommended

Full-stack framework with built-in API routes for RSVP → Google Sheets. Deploys to Vercel in one click from GitHub. Mobile-first CSS with smooth scroll animations. Best Vercel integration, largest community.

Pros

  • Native Vercel support
  • API routes = no separate backend
  • React ecosystem for animations

Cons

  • Heavier than plain HTML
  • Requires npm/build step
B

Plain HTML/CSS/JS + Vercel Serverless Function

Static HTML pages with vanilla JS for countdown/animations. One serverless function handles RSVP. No framework, no build step.

Pros

  • Simplest possible setup
  • Fastest load time
  • No framework to learn

Cons

  • More manual CSS work
  • Harder to manage as complexity grows
C

Astro + Vercel

Static site generator — outputs pure HTML with zero JS by default, adds JS only where needed. Excellent performance. Vercel serverless for RSVP API.

Pros

  • Best performance (minimal JS)
  • Great for content-heavy pages

Cons

  • Smaller community than Next.js
  • Less familiar syntax
