Design — Deployment & GitHub

How the project is versioned and hosted.

GitHub Setup

1. Create a GitHub repo (public or private)
2. Push the Next.js project
3. Add .env.local to .gitignore — secrets stay local
4. On Vercel: connect the GitHub repo → auto-deploy on every push to main

Environment Variables

.env.local (local) / Vercel Dashboard (production)
GOOGLE_SERVICE_ACCOUNT_EMAIL=xxx@yyy.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----..."
GOOGLE_SHEET_ID=your-spreadsheet-id

Deploy Workflow

Edit code locally
↓
git push origin main
↓
Vercel auto-builds & deploys (~30 sec)
↓
Live at your-project.vercel.app
(optionally connect a custom domain)

Performance target

Vercel's CDN + Next.js image optimization handles 30+ simultaneous opens easily. Static sections are served from the edge; only the RSVP POST hits a serverless function.
